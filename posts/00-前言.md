# 當 AI 已經人人在用，我這個 PM 也想紀錄下這段路

現在打開任何一個社群、任何一場分享會，幾乎都會聽到「AI」這兩個字。寫文案用 AI、做簡報用 AI、寫程式用 AI——好像一夜之間，人人都變成了 AI 的使用者。

但多數的分享，停在「這個 prompt 要怎麼下」「這個工具多厲害」。很少有人從一個**完全不懂技術的 PM 角度**，誠實地記錄下：跟一個 AI agent 一起整理工作，會踩到哪些坑、會學到哪些原來該懂卻一直沒搞懂的東西。

我就是那個 PM。不寫程式，過去對 git、repo、GitHub 這些字眼的認知大概停留在「工程師在用的東西」。但工作累積久了，手上的專案資料散落在各個資料夾裡，有的是履歷、有的是案子文件，有的甚至自己都忘了當初為什麼會建這個資料夾。

於是我開始用 Claude Code，想說既然要整理，就讓 AI agent 陪我一起搞懂這些東西到底是什麼、該怎麼用。

這篇前言想說的不是「我變強了」，而是想留下一段**真實的摸索軌跡**——包括看不懂術語時的尷尬、發現自己不小心把 GitHub Token 洩漏出去時的那種心跳加速、還有後來慢慢建立起一套屬於自己的工作方式時，那種「原來我也可以」的踏實感。

接下來這個系列，我會依照實際發生的順序，分享六個具體的心得：

1. PM 為什麼要懂一點 Git／GitHub
2. 手上明明有 7 個專案，資料夾卻亂成一團——怎麼整理
3. 我不小心把 GitHub Token 寫死在設定裡——一次資安教訓
4. 一個案子一個資料夾不夠用——用「範本倉庫」管理重複流程
5. 讓所有專案共用同一個「簡報 agent」
6. 回顧：PM 學 Claude Code，到底學到了什麼

如果你也是 PM，也想知道「不懂技術的人，怎麼跟 AI agent 一起把工作整理得更有條理」，歡迎跟我一起走這趟路。
