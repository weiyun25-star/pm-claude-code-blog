# 手上明明有 7 個專案，資料夾卻亂成一團

## 那句「這個文件你不是早就做好了嗎」，把我問倒了

某次跨部門會議上，主管臨時問了一句：「你之前那個保險案子的需求文件呢？先丟給我看一下。」

我嘴上說「好，我找一下」，手卻在電腦裡開始亂翻。`project` 資料夾、`complete-course` 資料夾、還是哪個案子順手存的？翻了快十分鐘，找到的還不確定是不是最新版。會議室裡的沉默，比文件本身還難找。

那一刻我才意識到：問題不是「我沒做這份文件」，是我**連自己手上有什麼、放在哪裡，都說不清楚**。

回去之後，我順手請 AI agent 幫我查一下，自己 GitHub 帳號底下到底有多少倉庫。結果列出來又嚇了一跳：`pm-workspace`、`insurance`、`learnhub`、`coffee-personality-quiz`、`resume-grace`、`weeklybm_tool`、`practice`——七個。

更尷尬的是，我本機的資料夾完全對不起來。有些專案根本沒有出現在任何一個看得懂的地方，有些則是被塞進一個叫 `complete-course` 或 `project` 的「雜物資料夾」裡，巢狀好幾層，名稱也跟倉庫名稱不一致。例如 `insurance` 這個倉庫——對，就是主管問的那個保險案子——本機對應的資料夾其實藏在 `project/insurance` 裡；`learnhub` 藏在 `complete-course/learnhub`；甚至同一個專案還出現兩份不同大小寫、內容也不完全一樣的資料夾（`iOPENMall` 跟 `iOPENMALL`）。

我心裡只有一個念頭：這樣下去，下次不管是我自己、同事，還是請 AI agent 幫忙找東西，都會卡在同樣的尷尬裡。

## 過程：先盤點，再決定怎麼歸位

整理的邏輯很單純，分三步：

1. **盤點 GitHub 上實際有哪些倉庫**——七個都列出來，當作對照表的「正解」。
2. **盤點本機資料夾，找出哪些其實是被 Git 追蹤的獨立倉庫**——用前一篇學到的方法（檢查有沒有 `.git`、看 remote 連到哪裡），一個一個確認。結果發現好幾個倉庫的本機資料夾，藏在跟倉庫名稱完全不相關的位置。
3. **規劃一個統一的位置，資料夾名稱直接對應倉庫名稱**——最後決定統一搬到 `/workspace/repos/<倉庫名稱>` 底下，例如 `repos/insurance`、`repos/learnhub`、`repos/pm-workspace`。

過程中還遇到一個意外：有個資料夾（`iOPENMall`）怎麼搬都搬不動，系統一直回報「裝置忙碌中」。後來才發現，它其實不是普通資料夾，而是一個獨立的磁碟掛載點——這種情況就不該硬搬，維持原地就好。這也是請 AI agent 一起整理的好處：它能順手檢查清楚「為什麼搬不動」，而不是直接用更暴力的方式硬上。

還有一些資料夾完全沒有對應的 GitHub 倉庫（例如裡面放著某個專案的 PM 文件、截圖、架構圖），這種「孤兒資料夾」我沒有照單全收地刪除或合併，是一個一個看內容、確認跟哪個專案有沒有關係，才決定保留、刪除，還是先不處理。

## 步驟教學：怎麼盤點自己的資料夾跟倉庫對不對得起來

如果你也想檢查自己的資料夾是不是也有同樣的問題，可以用對話請 AI agent 幫你做：

1. 先請它列出 GitHub 上的倉庫清單：
   > 「幫我看一下我 GitHub 帳號底下有哪些倉庫」
2. 接著請它檢查本機某個資料夾，是不是其中一個倉庫的本機副本：
   > 「幫我檢查一下這個資料夾是不是 git 倉庫，如果是的話，它連到 GitHub 上的哪個倉庫」
3. 如果手上有好幾個看起來亂糟糟、巢狀很深的資料夾，可以一次請它整理：
   > 「幫我盤點這幾個資料夾底下，有哪些是獨立的 git 倉庫，分別對應 GitHub 上的哪個倉庫，幫我整理一個對照表」
4. 確認對照表之後，再請它幫你搬到統一的位置：
   > 「幫我把這些對應到 GitHub 倉庫的資料夾，搬到統一的資料夾底下，並改成跟倉庫名稱一樣」
5. 如果有搬不動的資料夾，先別急著要求它用更強硬的方式處理，問清楚原因：
   > 「這個資料夾搬不動，幫我查一下原因」

走完這五步，手上的資料夾會跟 GitHub 上的倉庫一一對應，看資料夾名稱就知道它連到哪裡。

## 心得

這篇想強調的不是「資料夾要整理得多漂亮」，而是一個更現實的理由：

> PM 要橫跨好幾個案子追蹤進度、找文件、交接給別人。如果連自己的資料夾都理不清楚，要怎麼讓別人（甚至 AI agent）幫你快速找到正確的東西？

PM 常常是團隊裡唯一同時碰好幾個專案的人——工程師通常只專注自己負責的那個倉庫，但 PM 要同時追蹤多個案子的進度、找文件、隨時準備交接。如果連自己的資料夾結構都說不清楚，不管是請同事接手、還是請 AI agent 幫忙，都會卡在「先搞懂你的資料夾長怎樣」這一步，浪費掉本來可以拿來做事的時間。

「資料夾命名 = 倉庫名稱」這個原則很簡單，但效果立竿見影：以後不管是你自己、同事、還是 AI agent，看到資料夾名稱就知道它對應哪個倉庫，不會再卡在「這個改動要 push 去哪裡」這種小事上。
