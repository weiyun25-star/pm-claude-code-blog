# 讓所有專案共用同一個「簡報 agent」

## 每個案子都要做簡報，但範本只有一份

範本倉庫搞定之後，我發現還有一件事一直重複發生：不管手上是哪個案子，到了要對外報告的階段，幾乎都要做一份簡報。而且公司有固定的簡報範本格式，每次都要照著套。

問題是：這份範本不屬於任何一個案子，它是**所有案子都要共用**的東西。如果照前面文章的邏輯，把它複製一份塞進每個案子的資料夾裡，那以後範本改版，就要回頭一個一個案子去更新，根本不現實。

我心裡想的畫面是：以後跟 AI agent 說「幫我做簡報」，它會主動問我「要用哪一版範本？」，然後直接套用那份範本的格式產出——範本只存在一個地方，所有案子都認得到、用得到。

## 想讓所有案子共用一個 agent，應該怎麼做？

**共用工具的做法是建立一個獨立 Git 倉庫，讓所有案子的 CLAUDE.md 透過路徑引用——不複製、不各自維護。範本改版只需改一個地方，所有案子自動使用最新版本。**

決定方向之後，做法跟之前案子用的範本倉庫邏輯不一樣——**這次不是 clone 一份去客製化，而是建一個大家都直接引用、不複製的共用倉庫**：`presentation-templates`。

結構長這樣：

```
presentation-templates/
├── README.md
├── templates/
│   ├── MANIFEST.md          範本清單，記錄每一版的用途
│   └── v3-corporate-2026/   實際的範本檔案（pptx）
└── agents/
    └── presentation-agent.md   簡報 agent 的行為說明
```

`MANIFEST.md` 是關鍵：裡面用表格記錄每一版範本的版本號、資料夾、用途、適合場合，實際內容大概像這樣：

| 版本 | 資料夾 | 說明 | 適合場合 |
|------|--------|------|----------|
| V3 | `v3-corporate-2026/` | 公司簡報範本 (2026) | |

每新增一版範本，就在這個表格多加一行——`presentation-agent.md` 的行為很簡單：先讀這份清單列出可選版本、問使用者要用哪一版、讀取對應資料夾的範本、再套用格式產出簡報。

接下來要解決的問題是：**其他案子的 AI agent 怎麼知道要去找這個倉庫？** 答案不是把這個 agent 的內容複製進每個案子，而是在每個案子的 `CLAUDE.md` 裡，加一行「引用路徑」就好：

> 做簡報：「參考 @/workspace/repos/presentation-templates/agents/presentation-agent.md 幫我做簡報」（共用範本庫，不要複製進本專案）

這跟前一篇講的「案子專屬 agent」（像 `spec-reviewer.md`，每個案子各自複製一份方便客製化）是完全不同的處理方式——**案子專屬的東西複製，所有案子共用的東西引用**，這是判斷該怎麼存放一個工具時的關鍵分類。

## 步驟教學：怎麼建一個跨專案共用的工具

如果你也有類似「所有案子都要用、但不該各自複製一份」的東西（範本、共用規範、共用素材），可以這樣做：

1. 先請 AI agent 幫你把這個東西獨立成一個新倉庫：
   > 「幫我在 GitHub 上開一個新的倉庫叫 [名稱]，這個東西所有專案都要共用，不要放進任何一個案子裡」
2. 把實際的檔案（範本、規範文件）放進去，並請它整理一份清單說明每個項目的用途：
   > 「幫我把這些檔案分類整理，並寫一份清單說明每一個是做什麼用的」
3. 如果需要讓 AI agent 自動套用這個東西，請它寫一份說明這個共用工具該怎麼被使用：
   > 「幫我寫一份說明，告訴 AI agent 要怎麼讀取這個清單、詢問使用者要用哪一個、再套用」
4. 回到每個需要用到它的案子，請它在 `CLAUDE.md` 裡加一行引用，而不是複製整份內容過去：
   > 「幫我在這個案子的 CLAUDE.md 裡加一行，引用剛剛那個共用倉庫的 agent」
5. 之後在任何案子裡，只要說「參考 @[共用倉庫路徑]/agents/xxx.md，幫我做 xxx」，就能用到同一份、隨時最新的版本。

**PM學習小心得**

這篇真正的重點不是「怎麼做簡報」，而是學會判斷：**一個工具或範本，該複製給每個案子各自客製化，還是該集中維護、讓大家引用**。

判斷標準不是「現在內容是不是一樣」，而是：**這個東西未來會不會因為不同案子而需要長成不一樣的版本**。

- 像 `spec-reviewer.md` 這類案子專屬 agent，雖然一開始長得一樣，但每個案子的產品、利害關係人、技術限制都不同，三角色 review 的重點很可能需要因案子調整——如果共用同一份引用，為了 A 案子調整的內容會連帶影響到 B 案子，這是不該發生的。所以這種「會因案子需求而分裂演化」的東西,用**複製**，讓每個案子各自擁有一份、互不影響。
- 像簡報範本，所有案子該套用完全一樣的格式，沒有「這個案子要長不一樣」的需求。各自複製反而是負擔——範本改版要追著每個案子去更新。這種「不該分裂、永遠該長一樣」的東西，才適合**集中維護、用引用共用**。

搞清楚這個分類之後，以後團隊裡任何「大家都要用到的東西」，就知道該往哪裡放，不會讓多專案協作越做越亂。

## 常見問題（FAQ）

### 工具「複製」和「引用」有什麼差別？
複製是把工具檔案塞進每個案子的資料夾，之後各自獨立修改，互不影響。引用是所有案子的 CLAUDE.md 都記著同一個共用倉庫的路徑，工具有更新時只要改那一個地方，所有案子自動使用最新版。

### 什麼樣的工具該複製？什麼該引用？
判斷標準是：**這個工具未來會不會因不同案子而需要長成不同版本？** 會，就複製；不會，就引用。spec-reviewer 會隨案子特性調整，所以複製；公司簡報範本格式固定，所以引用。

### 在 CLAUDE.md 裡加引用路徑，AI 一定找得到嗎？
只要路徑正確、共用倉庫在本機已經 clone 下來，AI agent 就能讀到。如果路徑不存在，AI 會回報找不到檔案，這時候先確認共用倉庫有沒有 clone 到本機的對應位置。

### 共用倉庫改版了，需要通知所有用到它的案子嗎？
不需要，這正是引用而非複製的優點。各案子的 CLAUDE.md 裡寫的是路徑，只要路徑對到的那份檔案更新了，下次 AI 讀取時就是最新版，不用逐一通知或手動更新。

### MANIFEST.md 一定要做嗎？
建議做。當共用倉庫裡有多個版本或多個工具時，MANIFEST.md 是讓 AI agent 知道「有哪些版本可選、各版本用在什麼場合」的索引表。少了它，AI 只能猜或每次都問你要用哪一份。
