# 一個案子一個資料夾不夠用——用「範本倉庫」管理重複流程

## 既然都聽到要做成範本了，那我就自己做一個

這段時間我一直聽到身邊的人在講「重複的事要做成 skill」「這個流程應該寫進 CLAUDE.md，不要每次重新講一遍」。一開始聽不太懂，只覺得這些詞好像很厲害，但跟我一個 PM 有什麼關係？

後來資料夾整理好之後，我開始想下一個問題：以後接到新案子，難道每次都要重新生出一套 BRIEF、訪談紀錄、Spec 的資料夾結構？每次都要重新跟 AI 解釋一次「我要的格式是這樣」？這時候才突然意識到——這不就是大家說的「重複的事要做成範本」嗎？

於是我沒有再等，直接動手請 AI agent 幫我把自己的工作流程整理成一份**範本倉庫**——`pm-workspace`。不是去找現成的工具來套，而是照著我自己接案子的習慣，一步一步把該有的東西生出來。

## 過程：我建出來的範本結構長什麼樣子

整理完之後，這份範本大概是這樣：

```
pm-workspace/
├── CLAUDE.md       全局規則：語言、檔案存放規則、開始新案子前要做什麼
├── BRIEF.md        案子背景空白範本（目標用戶、成功定義、利害關係人...）
├── agents/         專門用的 agent（review、估點、會議整理...）
├── templates/      Spec、會議紀錄的空白模板
├── knowledge/       我認同的方法論
├── context/        my-role.md，讓 AI 知道我是誰、工作習慣是什麼
└── output/
    ├── requirements/
    ├── specs/
    ├── meeting-notes/
    └── deliverables/
```

重點不是資料夾本身，而是**每個資料夾都對應一條明確的規則**：訪談紀錄固定存哪裡、Spec 固定存哪裡、開始新案子前要先讀哪個檔案、工作階段結束前要更新哪個區塊。這些規則寫進 `CLAUDE.md` 之後，AI agent 每次跟我配合，都照同一套邏輯走，不用我每次重新講一遍。

建好範本之後，下一個問題是：**新案子要怎麼套用這份範本？** 答案不是把範本內容複製貼上，而是用 git 的方式整份「clone」過去，這樣新案子一開始就帶著完整的結構跟規則，後面才開始累積自己案子專屬的內容。

完整的使用流程是這樣：

1. **clone 一份 pm-workspace**：到新資料夾，這時候內容跟 pm-workspace 一模一樣，帶著現成的 `agents/`、`templates/`、`BRIEF.md` 結構。
2. **在 GitHub 上開一個全新的空倉庫**：用這個案子的名字（例如 `insurance`）。
3. **把這個資料夾的 remote 改成新倉庫的網址**：不然之後做的修改，會一直想推回 `pm-workspace` 本身，而不是這個案子自己的倉庫。
4. **push 上去**，把範本結構帶到新倉庫，之後照 `CLAUDE.md` 的規則開始填 `BRIEF.md`、累積這個案子專屬的內容。

## 步驟教學：怎麼用範本倉庫開一個新案子

如果你也想建立一份自己的範本、用同樣的方式管理重複性的工作，全程用對話就能完成：

1. 先請 AI agent 幫你把目前的工作流程整理成範本：
   > 「我每次接案子都會做這些事：[列出你的固定流程]，幫我整理成一份可以重複使用的範本資料夾結構」
2. 範本建好之後，要開新案子時說：
   > 「幫我把這份範本 clone 一份，建立一個叫 [案子名稱] 的新資料夾」
3. 接著請它確認/建立對應的新倉庫，並把連線改過去：
   > 「幫我在 GitHub 上開一個新的倉庫叫 [案子名稱]，然後把這個資料夾的 remote 改成連到這個新倉庫」
4. 確認連線正確後，請它推上去：
   > 「幫我把這個資料夾 push 上去」
5. 之後開始這個案子，直接說：
   > 「我接到一個新案子，幫我問清楚背景，填到 BRIEF.md」

   它會照範本裡寫好的規則，問你產品、目標用戶、時程資源等問題。

## 心得

這篇最大的收穫，是體會到「做成範本」這件事，門檻其實沒有想像中高——不需要先學會什麼厲害的工具或框架，**先把自己手上重複在做的事情，誠實地列出來，再請 AI agent 幫你整理成一份可以重複使用的結構，就已經是「做成範本」了**。

對其他 PM 的建議：工作流程裡「重複性高」的部分（像是每次接案都要做的固定動作），值得花時間做成一份範本，讓 AI agent 接手執行固定步驟，而不是每次都憑印象重新來一次。下一篇我會更深入拆解這份範本裡實際放了哪些規則、哪些 agent，怎麼組合起來變成一整套自己的工作方法論。
