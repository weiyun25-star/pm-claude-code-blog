# PM 學習 Claude Code 部落格系列

紀錄一位 PM（不寫程式、非工程背景）跟 AI agent（Claude Code）一起整理工作、處理多專案協作的真實過程。給其他 PM／非工程背景的人看，語氣專業、結構化，不堆術語。

## 系列大綱

| 篇號 | 標題 | 核心心得 |
|------|------|----------|
| 00 | 當 AI 已經人人在用，我這個 PM 也想紀錄下這段路 | 前言，感性語氣，系列入口 |
| 01 | PM 為什麼要懂一點 Git／GitHub？ | Git／Repo／GitHub 的基本類比 |
| 02 | 手上明明有 7 個專案，資料夾卻亂成一團 | 資料夾命名 = 倉庫名稱，多專案協作不打結 |
| 03 | 我不小心把 GitHub Token 寫死在設定裡 | 資安意識不是工程師專屬 |
| 04 | 一個案子一個資料夾不夠用——用「範本倉庫」管理重複流程 | 模板化重複性工作 |
| 05 | 讓所有專案共用同一個「簡報 agent」 | 案子專屬 vs 全專案共用工具的分類判斷 |
| 06 | 回顧——PM 學 Claude Code，到底學到了什麼 | 總結 5 個心得＋給其他 PM 的建議 |
